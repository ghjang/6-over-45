# 로또 6/45 프로젝트 개선 제안서 (Updated)

## 1. 프로젝트 현황 분석 (Updated)
- **구조**:
    - **Data**: `6-over-45` (Python 스크립트, GitHub Actions).
    - **Frontend**: `s-comp-box` 리포지토리 내의 `webapp/6-over-45` (SvelteKit).
    - **Library**: `s-comp-core` (Svelte 컴포넌트 라이브러리, Frontend가 의존).
- **배포**: `6-over-45`의 GitHub Action이 `s-comp-box`를 체크아웃하여 빌드 후 배포.
- **로직**: 과거 당첨 번호의 빈도(Frequency)를 가중치로 사용하여 랜덤 추출.

## 2. 개선 제안

### ⚙️ 리포지토리 및 인프라 통합 (Priority)
현재 데이터와 프론트엔드 코드가 분리되어 있어 관리가 복잡합니다. `s-comp-box`는 여러 컴포넌트와 앱이 섞인 "Box" 성격의 리포지토리이므로, 로또 앱만 분리하여 통합하는 것이 바람직합니다.

- **웹앱 코드 이관**:
    - `s-comp-box/webapp/6-over-45` 폴더를 이 리포지토리의 `web/` 폴더로 이동.
    - `s-comp-core`는 npm 의존성으로 유지 (현재와 동일).
- **GitHub Actions 수정**:
    - 외부 리포지토리(`s-comp-box`) 체크아웃 단계를 제거.
    - 내부 `web/` 폴더에서 빌드하도록 워크플로우 수정.
    - 데이터 파일(`data/*.json`) 주입 경로를 로컬 경로로 단순화.

### 💡 알고리즘 및 로직 (Logic)
- **Hot & Cold 전략 추가**:
    - **Hot**: 최근 5~10회차 내 자주 등장한 번호 위주 추천.
    - **Cold**: 오랫동안 나오지 않은 번호(미출현 번호) 위주 추천.
- **패턴 분석 도입**:
    - **연속 번호 배제/포함**: 로또 번호는 의외로 연속 번호(예: 12, 13)가 자주 등장합니다. 이를 반영하거나 배제하는 옵션.
    - **구간별 분포**: 1~10번대, 11~20번대 등 구간별로 고르게 분포되도록 필터링.
    - **홀짝/합계 비율**: 홀수/짝수 비율이 4:2, 3:3, 2:4인 조합만 필터링.

### 🎨 UI/UX (Frontend)
`s-comp-core` 라이브러리를 활용하거나 커스텀 컴포넌트를 추가하여 시각적 요소를 강화합니다.

- **시각적 요소 강화**:
    - **로또 공 디자인**: 실제 로또 공 색상(1~10 노랑, 11~20 파랑, 21~30 빨강, 31~40 검정, 41~45 초록) 적용.
- **재미 요소 추가**:
    - **"지난주 결과 확인"**: 추천해줬던 번호가 실제로 몇 등이었는지 시뮬레이션 결과 표시.
    - **인터랙션**: 번호 추첨 시 슬롯머신 효과 또는 `s-comp-core`의 `ScrollingText`/`Marquee` 등을 활용한 애니메이션.

### 🛡️ 데이터 신뢰성
- **API 활용 고려**: 현재 `dhlottery.co.kr` HTML 파싱 방식은 사이트 개편 시 작동이 멈출 수 있음. (공식 API가 없다면 현재 방식 유지하되, 파싱 실패 시 알림 기능 강화).
